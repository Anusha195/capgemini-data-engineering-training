# SQL Analytics & Query Techniques – Week 2 Day 1

## Objective  
The objective of this assignment is to understand and implement SQL concepts including subqueries, Common Table Expressions (CTEs), window functions, and analytical calculations such as Month-over-Month (MoM) analysis and customer behavior evaluation.

---

## Files Included  

- Subquery_CTE.sql – Contains problems and solutions using subqueries and CTEs  
- Lead_lag.ipynb - Analytics_WindowFunctions (Contains analytical queries using window functions)  

---

## Dataset Description  

The assignment uses sample relational tables:

- Employees – Stores employee details such as salary, department, and manager  
- Departments – Contains department information  
- Orders – Stores order transactions linked to employees  

These tables are created and populated with sample data for query execution and analysis.

---

## Key Concepts Covered  

### 1. Subqueries  
- Filtering employees based on average salary  
- Using scalar subqueries for minimum and maximum comparisons  
- Using EXISTS to identify employees with orders  
- Using ALL operator for advanced comparisons  

### 2. Common Table Expressions (CTEs)  
- Creating reusable query blocks using WITH clause  
- Performing aggregations inside CTEs  
- Joining multiple tables within CTEs  
- Comparing employee salary with department averages  

### 3. Window Functions  
- LAG() to access previous row values  
- LEAD() to access next row values  
- RANK() to assign rankings  
- PARTITION BY to segment data  

### 4. Analytical Calculations  

#### Month-over-Month (MoM) Analysis  
- Extracted month from order date  
- Calculated total monthly sales  
- Used LAG() to get previous month sales  
- Computed:
  - Absolute change in sales  
  - Percentage change in sales  

#### Customer Loyalty Analysis  
- Calculated time difference between consecutive orders using LEAD()  
- Used DATEDIFF() to compute gap between orders  
- Calculated average days between orders  
- Ranked customers based on ordering frequency  

---

## Issues Identified  

- Null values generated by LEAD() for the last record  
- Ranking impacted by null values  
- Need for proper ordering in window functions  
- Possible divide-by-zero errors in percentage calculations  

---

## Data Handling Steps  

- Used COALESCE() to handle null values  
- Applied ORDER BY correctly in window functions  
- Used CAST for accurate percentage calculations  
- Applied ROUND for better readability  

---

## Validation  

- Verified intermediate query outputs step-by-step  
- Cross-checked aggregated values such as monthly sales  
- Ensured ranking logic is correct  
- Validated null handling  

---

## Key Learnings  

- Window functions are essential for time-based and sequential analysis  
- LAG() and LEAD() simplify comparison between rows  
- CTEs improve readability and structure of complex queries  
- Subqueries are useful but less readable compared to CTEs  
- Proper handling of nulls is critical in analytical queries
